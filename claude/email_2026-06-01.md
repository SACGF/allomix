WENDY'S DONOR HOM SITE - ANY HOST PRESENCE DETECTION 

* Wendy described a new method - where the donor is homozygous (HOM REF or HOM ALT), the presence of the donor-absent allele (the one the host carries but the donor lacks) is evidence the host is present. This is a great addition to the Maximum Likelihood Estimation (MLE) model, and can help at very low LOD, but depends on the error background. At clean error and good depth it actually beats the MLE (LoD 1.5-7x lower in our sims); at high error it degrades fast (worst case, a sibling pair at shallow 1000x depth and 1% error, LoD collapses to ~14%; deeper coverage holds ~0.2-1% even at 1% error). So we need to be super careful about sequencing errors, index hopping etc - but I think we can work those out and we should use both.

It is plotted in run2_run3_run10_presence.png alongside the MLE. The two answer different questions: the MLE estimates how much donor/host there is (a fraction) across all informative markers; the presence test asks a single yes/no - is the host there at all? - and returns a p-value. It uses only the donor-homozygous markers, and because the donor-absent allele should sit at the sequencing-error floor in a pure-donor sample, it is freed from the two things that blunt the MLE at the low end (the shared overdispersion term and the global error rate). That is why it can beat the MLE at clean error / high depth, with the trade-off being error sensitivity. The two also cross-check each other: the presence estimate should track (1 - donor fraction), so if presence says "host is here" while the MLE says ~100% donor, that disagreement is itself a QC flag worth a look.

I plotted this "Per-marker host-presence structure - run 9" graph (run 9 = full panel, pre-filter). You can see that a few locations are very high. I wanted to see whether this was real or an artifact (mismapping, seq errors, etc) - so made a Manhattan plot. VCF CALLING ISSUES

Wendy pointed out that the panel I used had been trimmed to be close to exons, missing some intron markers - switched to using the full panel, gaining us more markers.

I had to modify the VCF calling slightly - the basics are:

* Joint call host/donor germline
* Look at allele depth at those sites in the admixture sample

But there were complexities:

* GATK sometimes holds back on a germline call thinking it's an error, but if you pass enough samples in, it can see that it's real - so I supplied gnomAD common (>5% AF) sites as known sites to the joint caller. This has rescued some borderline HET calls and gains us a few more markers.
* GATK has filters on very rare AD calls (as do almost all variant callers, thinking them errors) which wrecks our very low-LOD. Solution is to use "pileup" which just gives raw allele depth counts rather than any sophisticated variant calling / filtering
